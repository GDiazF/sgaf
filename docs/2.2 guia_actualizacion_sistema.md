# Guía 2.2: Actualización del Sistema desde GitHub

Esta guía detalla los pasos para actualizar el servidor de producción con la última versión del código, garantizando la seguridad de los datos.

## 1. PASO PREVIO: Respaldo de seguridad
Nunca actualices sin tener una copia fresca de los datos. Ejecuta estos comandos en la terminal de Ubuntu:

```bash
# Respaldo de Base de Datos
docker exec sgaf_db_prod pg_dump -U sgaf_user sgaf_db > ~/respaldo_preventivo.sql

# Respaldo de Media (Fotos/Archivos)
cp -r ~/sgaf/backend/media ~/media_preventivo
```

## 2. DESCARGA: Obtener el nuevo código
Entra a la carpeta del proyecto y descarga los cambios de GitHub:

```bash
cd ~/sgaf
git pull origin reingenieria
```

### ⚠️ Solución de Conflictos Comunes
Si el `git pull` falla por archivos como `.email_counter.json`, ejecuta:
```bash
# Limpiar cambios locales automáticos
git checkout -- backend/solicitudes_reservas/.email_counter.json
# Intentar el pull de nuevo
git pull origin reingenieria
```

## 3. DESPLIEGUE: Aplicar los cambios
Para que el nuevo código empiece a funcionar, debemos reconstruir los contenedores:

```bash
docker compose up -d --build
```
*Este comando reemplaza el código pero **mantiene intactos** tus datos de la base de datos y la media.*

## 4. VERIFICACIÓN: Usuarios y Acceso
Si necesitas verificar que los usuarios existen o si olvidaste la contraseña:

**Ver lista de usuarios:**
```bash
docker exec -it sgaf_backend_prod psql -U sgaf_user -d sgaf_db -c "SELECT username FROM auth_user;"
```

**Cambiar contraseña de administrador:**
```bash
docker exec -it sgaf_backend_prod python manage.py changepassword admin
```

---
> [!IMPORTANT]
> Si el sistema no responde después del build, revisa los logs con: `docker compose logs -f backend`

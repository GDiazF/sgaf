# Guía de Configuración para Desarrollo Local - SGAF

Este documento describe los pasos necesarios para levantar el sistema SGAF en un entorno de desarrollo local (Windows/macOS/Linux) sin utilizar Docker.

## 1. Requisitos Previos

Asegúrate de tener instalados los siguientes componentes:
- **Python 3.10 o superior**
- **Node.js 18 o superior** (incluye npm)
- **Git**

---

## 2. Preparación del Backend (Django)

El backend gestiona la lógica de negocio y la base de datos SQLite.

1.  **Navegar a la carpeta del backend:**
    ```powershell
    cd backend
    ```

2.  **Crear y activar un entorno virtual:**
    ```powershell
    # Crear entorno
    python -m venv venv

    # Activar (Windows)
    .\venv\Scripts\activate

    # Activar (Linux/macOS)
    source venv/bin/activate
    ```

3.  **Instalar dependencias:**
    ```powershell
    pip install -r requirements.txt
    ```

4.  **Configurar Variables de Entorno (.env):**
    Crea un archivo llamado `.env` dentro de la carpeta `backend/` con el siguiente contenido base:
    ```env
    DEBUG=True
    SECRET_KEY=django-insecure-local-dev-key
    ALLOWED_HOSTS=localhost,127.0.0.1
    CORS_ALLOWED_ORIGINS=http://localhost:5173
    CSRF_TRUSTED_ORIGINS=http://localhost:5173
    ```
    *Nota: Al no definir `DB_NAME`, el sistema usará SQLite automáticamente.*

5.  **Base de Datos (SQLite):**
    Asegúrate de que el archivo `db.sqlite3` se encuentre en la raíz de la carpeta `backend/`. Si es una instalación nueva, puedes generar las tablas con:
    ```powershell
    python manage.py migrate
    ```

6.  **Iniciar Servidor de Desarrollo:**
    ```powershell
    python manage.py runserver 8000
    ```
    El API estará disponible en `http://localhost:8000/api`.

---

## 3. Preparación del Frontend (Vite + React)

El frontend se comunica con el backend mediante peticiones HTTP.

1.  **Navegar a la carpeta del frontend:**
    ```powershell
    cd frontend
    ```

2.  **Instalar dependencias de Node:**
    ```powershell
    npm install
    ```

3.  **Configurar Variables de Entorno (.env):**
    Crea un archivo llamado `.env` dentro de la carpeta `frontend/` para indicar la ruta del API:
    ```env
    VITE_API_URL=http://localhost:8000/api
    ```

4.  **Iniciar Servidor de Desarrollo:**
    ```powershell
    npm run dev
    ```
    La interfaz estará disponible en `http://localhost:5173`.

---

## 4. Resumen de Flujo de Trabajo

Para trabajar diariamente en el sistema:
1.  **Terminal 1 (Backend):** `cd backend` -> `.\venv\Scripts\activate` -> `python manage.py runserver`
2.  **Terminal 2 (Frontend):** `cd frontend` -> `npm run dev`

Ambos servicios deben estar corriendo simultáneamente para que el sistema funcione correctamente.

# Guía de Despliegue y Orquestación Docker - SGAF

Este manual describe el proceso para desplegar el sistema SGAF desde un repositorio de Git hasta un servidor de producción.

## 1. Descarga del Código
Desde el directorio home del usuario:
```bash
git clone -b reingenieria https://github.com/GDiazF/sgaf.git
cd sgaf
```

## 2. Configuración del Ambiente (.env)
El sistema utiliza un archivo `.env` en la raíz para separar las credenciales del código fuente.
```bash
nano .env
```

### 🔧 Configuración utilizada (Ejemplo para IP 10.0.100.99):
> [!WARNING]
> ⚠️ **ATENCIÓN AL CAMBIAR SERVIDOR:** Debes reemplazar obligatoriamente `10.0.100.99` por la nueva IP de tu servidor en TODAS las variables de red de este bloque (`ALLOWED_HOSTS`, `CORS`, `CSRF` y `VITE_API_URL`).

Puedes usar este bloque como plantilla base:

```env
# --- IDENTIDAD DEL AMBIENTE ---
DEBUG=False
SECRET_KEY=django-insecure-z*#p5r!d1@9z9(s6-u*f-1o#p6_r8_m=u4#0r!v9t_=2t_1j_8

# --- RED Y SEGURIDAD ---
ALLOWED_HOSTS=10.0.100.99,localhost,127.0.0.1
CORS_ALLOWED_ORIGINS=http://10.0.100.99:8080
CSRF_TRUSTED_ORIGINS=http://10.0.100.99:8080

# --- BASE DE DATOS (POSTGRESQL EN DOCKER) ---
DB_NAME=sgaf_db
DB_USER=sgaf_user
DB_PASSWORD=sgaf_contrasenia_secreta_2026
DB_HOST=db
DB_PORT=5432

# --- URL DE LA API (FRONTEND COMUNICATION) ---
VITE_API_URL=http://10.0.100.99:8080/api

# --- LÍMITES ---
EMAIL_DAILY_LIMIT=200
```

## 3. Lanzamiento de Contenedores
Para construir y levantar el sistema:
```bash
docker compose up -d --build
```
*Esto levantará: `sgaf_db_prod`, `sgaf_backend_prod`, `sgaf_frontend_prod` y Nginx.*

## 4. Gestión de Datos (Restauración de Backup)
Para poblar el sistema con un respaldo (`.sql`) generado anteriormente:

1. Subir el archivo al servidor (vía SCP):
   `scp backup.sql user@ip:/home/user/`
2. Inyectar el respaldo en el contenedor:
   `cat ~/backup.sql | docker exec -i sgaf_db_prod psql -U sgaf_user -d sgaf_db`

## 5. Mantenimiento y Logs
Ver estado: `docker ps`
Ver logs: `docker compose logs -f`
Reiniciar: `docker compose restart`
